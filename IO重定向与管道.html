IO重定向与管道

IO重定向与管道

在linux中，一切东西都是文件。

每个文件都有一个文件描述符与之对应，这里在介绍IO重定向之前，先介绍三个特殊的文件描述符。

0 ---> stdin    # 标准输入
1 ---> stdout   # 标准输出
2 ---> stderr   # 标准错误

标准输出重定向

使用 > 来重定向标准输出到文件，相当于truncate模式，每次写入前会清空文件

echo "some content1" > file1.txt
cat file1.txt
echo "some content2" > file1.txt
cat file1.txt

使用 >> 来重定向标准输出到文件，相当于append模式，每次写入都会追加在之前的内容

echo "some content1" >> file2.txt
cat file2.txt
echo "some content2" >> file2.txt
cat file2.txt

标准错误重定向

使用 2> 来重定向标准输出到文件，相当于truncate模式，每次写入前会清空文件

ls -eee 2> error1.txt
cat error1.txt
ls -eee 2> error1.txt
cat error1.txt

使用 2>> 来重定向标准输出到文件，相当于append模式，每次写入都会追加在之前的内容

ls -eee 2>> error2.txt
cat error2.txt
ls -eee 2>> error2.txt
cat error2.txt

使用 2>&1 把标准错误重定向到标准输出

ls -eee >/dev/null 2>&1  # 把标准错误重定向到标准输出，然后把标准输出重定向到/dev/null
ls -eee &> /dev/null     # 等价于上面的写法

重定向的顺序问题

下面的两种写法顺序不同，含义也大相径庭。

ls -eee >/dev/null 2>&1 # 写法一
ls -eee 2>&1 >/dev/null # 写法二

写法一先将标准输出重定向到/dev/null黑洞文件，然后将标准错误输出重定向到标准输出(已经指向/dev/null文件)，结果就是标准输出和标准错误都指向/dev/null。 写法二先将标准错误指向标准输出(实际上是复制了文件描述符1)，然后将标准输出重定向到/dev/null，结果就是标准错误指向文件描述符1，而标准输出指向/dev/null

标准输入重定向

使用 < 来重定向标准输入

cat < error2.txt > error3.txt   #把cat命令的输入重定向到error2.txt文件，然后输出结果打印到error3.txt中

管道符号

管道符号很强大。linux有很多的命令，但是每个命令都只能完成特定的一类功能。因此往往需要把多个命令组合起来使用来达到我们的目的。 使用管道符号 | 可以串联多个命令。即把前面命令的输出作为后面命令的输入。

cat error2.txt | head -n 3  # cat命令的输出传入head命令，head命令截取其前三行

在上面的例子中，cat命令的输出通过管道符号传递到了head命令的标准输入，而head命令会从标准输入读取内容，因此完成了命令的串联。 但是，针对一些不直接读取标准输入，而是需要传入参数来执行的命令，如何通过管道来串联执行呢？ 答案是使用xargs命令。xargs命令是给其他命令传递参数的一个过滤器，它会把前面命令的输出作为后面命令的参数。

ls | xargs -i echo {}

上面的例子中，ls的输出通过管道传递到后面，xargs命令把传递过来的ls的结果作为参数传递给echo命令。{}是参数占位符，代表ls结果中的每一行。 使用-i选项时，默认是采用{}作为占位符。可以通过xargs命令的-I选项来显式指定参数占位符，下面的命令用[]作为参数占位符

ls | xargs -I [] echo []

除了 | 符号外，&|符号也可以作为管道符号，其会将前面命令的标准输出标准错误都作为后面命令的标准输入。它可以看做是 2>&1 | 的简写。下面两种写法等价。

ls -eee |& xargs -i echo {} > ttt.txt
ls -eee 2>&1 | xargs -i echo {} > ttt.txt

管道中的每个命令都是在新的子shell中执行的，因此对当前shell的变量不产生影响。

a=4
a=5 | echo $a       # 4

管道的返回值默认是管道中的最后一个命令的返回值。然而如果环境变量pipefail被设置了的话，则返回管道中的最后一个执行失败的命令的返回值，如果所有命令都执行成功，则返回0。

ls | ls -eee | ls
echo $?         # 0

set -o pipefail
ls | ls -eee | ls
echo $?         # 2

命令的组合

除了上面的管道之外，还有一些其他方法来将多个命令组合起来执行。

&&符号连接起来的多个命令在执行时会发生逻辑短路情况，只有前面的命令执行成功(退出状态为0)时才会执行后面的命令。

ls -eee &> /dev/null && echo "success"      # 无输出
ls -al &> /dev/null && echo "success"       # 输出success

||符号刚好和&&相反，连接起来的多个命令在执行时，只有当前面的命令执行失败时才会执行后面的命令。

ls -eee &> /dev/null || echo "failed"      # 输出failed
ls -al &> /dev/null || echo "failed"       # 无输出

&& 和 || 串起来的多个命令组合 其整体的返回值 取的是最后一条被执行的命令的返回值。

除了上面提到的2个命令组合方式之外，还有其他的命令组合方式，如;和&，他们的介绍放在shell之进程管理这一章中。
