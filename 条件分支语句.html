条件分支语句

条件分支语句

shell作为一种编程语言，内部也有条件与分支语法if/else，用来处理多分支流程。

if/else语法

if后的condition

下面是shell中的if/else语法对应的伪代码

if condition1 AND condition2; then
    do something
else if condition3 OR condition4; then
    do something
else if NOT condition5; then
    do something
fi

从上面的伪代码可以看出，shell的if语句中是是支持或、且、非三种逻辑组合的。

注意： shell的if语句只能对退出状态做测试。

因此，如果condition是命令，则可以直接放在if语句中：

if ls; then     # 如果ls执行成功，返回0(True)
    echo "ls success"
else
    echo "ls failed"
fi

如果希望if语句中对表达式的结果进行判断，如变量比较、文件状态测试，则需要借助以下两种等价的方式：

# 使用test命令来将表达式的值与if联用
if test "aaa" = "aaa" ; then     # 如果表达式 "aaa"="aaa"的值为True，返回0(True)退出值
    echo "true"
else
    echo "false"
fi

# 使用[]括号，来将表达式的值与if联用
if [ "aaa" = "aaa" ]; then # 注意中括号和表达式之间都要有空格，否则语法错误
    echo "true"
else
    echo "false"
fi
  1. 从上面的代码可以看出来，当把退出状态作为if的判断条件时，如果退出状态为0，表示True，执行if语句逻辑；否则如果退出状态非0，表示False，则执行else分支。这点跟其他编程语言刚好相反！
  2. if后面的condition中，中括号和表达式之间都要有空格，否则语法错误

字符串比较

如前所述，shell中所有变量均为字符串类型，因此字符串比较可以直接使用比较运算符。

表达式 含义
str1 = str2 如果str1和str2相同则为True
str1 != str2 如果str1和str2不相同则为True
str1 > str2 如果str1 > str2 则为True
str1 < str2 如果str1 < str2 则为True
-n str 如果str非null(长度大于0)则为True
-z str 如果str为null(长度为0)则为True

注意：

  1. 上述字符串大小比较时，针对>和<号，在实际使用时必须使用\进行转义，否则shell会误把其当成是重定向符号。
  2. 在做相等性比较时，=和==是等价的
if [ "aaa" \> "bbb" ]; then
    echo "aaa > bbb"
fi

if [ -z "" ]; then
    echo "empty str"
fi

数字比较

如前所述，shell中只有字符串变量，因此，如果想要进行数字大小的比较，不能简单的使用<和>符号，因为默认<和>是按照字符串格式来比较的。

表达式 含义
n1 -gt n2 数字n1大于n2
n1 -ge n2 数字n1大于或等于n2
n1 -eq n2 数字n1等于n2
n1 -lt n2 数字n1小于n2
n1 -le n2 数字n1大于或等于n2 
if [ 3 -gt 4 ]; then
    echo "imposible"
fi

-f file file为常规文件
-b file file为块设备文件
-c file file为字符设备文件

-S file file为socket文件

if条件逻辑组合

如前所述，if后面可跟多个condition，支持AND/OR/NOT等逻辑组合。

# AND语法1
if [ "3" \> "2" ] && [ "2" \< "1" ];then
    echo "true"
fi

# AND语法2
if [ "3" \> "2" -a "2" \< "1" ];then
    echo "true"
fi

# OR语法1
if [ "3" \> "2" ] || [ "2" \< "1" ];then
    echo "true"
fi

# OR语法2
if [ "3" \> "2" -o "2" \< "1" ];then
    echo "true"
fi

# NOT语法
if [ "3" \> "2" -o ! "2" \< "1" ]; then
    echo "true"
fi

if语句简化

利用&&和||的原理，可以对一些if语句简化。

[ -f file1 ] && cat file1   # 如果file1存在且为常规文件，则打印其内容
[ -e file1 ] || touch file1 # 如果file1不存在，则利用touch命令来创建

[ ] 和 [[ ]]的区别

条件判断语句中的condition刚才介绍了2种写法：test expr 和 [ expr ]。这两种写法是完全一致的。除此之外，还有一种shell内部支持的写法[[]]，即2个中括号，这个写法与刚才的两种写法在语法规则上面有较大的不同。

表达式 实质 特点
test和[] shell的内部命令。condition是作为参数传递的。大多数shell都支持 1.<和>需要转义，否则会被当做重定向符号。
2. condition可以整体放在双引号中，比如"4" "<" "3"，因为在传递参数时会自动去掉双引号
3. [ condition1 -a condition2 ] 没有逻辑短路
[[]] bash的关键字，只有一部分shell支持 1. 内部< 和 >不需要转义
2. 内部可以直接使用&& \		 \
3. [[ condition1 && condition2 ]] 会有逻辑短路
  • [[]]字符串比大小时不需要转义，下面2中写法一致
if [ "4" \< "3" ]; then
    echo "ttt"
fi

if [[ "4" < "3" ]]; then
    echo "ttt"
fi
  • [[]]支持在内部写 && 和 || 。下面三种写法一致
if [ "3" \> "2" -a "4" \> "3" ]; then
    echo "true"
fi

if [ "3" \> "2" ] && [ "4" \> "3" ]; then
    echo "true"
fi

if [[ "3" > "2" && "4" > "3" ]]; then
    echo "true"
fi
  • [[]]支持简单的字符串模式匹配。注意模式字符串必须放在等号右边，并且不加双引号。如果加了双引号，则会被当做普通字符串对待。这里的=和==等价。如果shell的nocasematch选项被打开，则匹配时不区分大小写
# 模式字符串必须放在等号右边，并且不加双引号
if [[ "abc" == a* ]]; then
    echo "bingo"
fi
  • [[]]支持字符串的正则表达式匹配。 这里要使用=~符号，模式字符串必须放在等号右边，并且不加双引号。如果加了双引号，则会被当做普通字符串对待。如果shell的nocasematch选项被打开，则匹配时不区分大小写
# 模式字符串必须放在等号右边，并且不加双引号
if [[ "/aaa/bbb/ccc" =~ /*c{3} ]]; then
    echo "bingo"
fi

使用=~进行正则匹配时，匹配结果会存储在数组变量BASH_REMATCH中。${BASH_REMATCH[0]}表示整个被匹配的字符串，${BASH_REMATCH[n]}代表模式中第n个小括号内所匹配到的部分。

针对上面的正则匹配，pattern字符串的引号和\需要给小心对待。在pattern中，双引号括起来的部分直接作为原始字符来匹配，而单引号括起来的部分则保留其在正则表达式中的原始含义。如'a.b'中的.表示匹配所有字符，而"a.b"中的.就只匹配.符号。

另外，pattern中的\用来对特殊字符进行转义，但是当pattern是存储在某个变量中的时候，就需要小心对待，否则会引起错误。

pattern='\.'

[[ . =~ $pattern ]]     # 成功匹配
[[ . =~ \. ]]           # 成功匹配

[[ . =~ "$pattern" ]]   # 匹配失败
[[ . =~ '\.' ]]         # 匹配失败

上面的代码中，前两个匹配成功，\符号对.符号进行了转义。 后两个匹配失败，原因是\也被当成是需要被匹配的字符了。

condition内空格的强调

注意： 无论对于test、[]、还是[[]]，变量和比较运算符以及外围的中括号之间都必须有空格！如果少了空格，可能报语法错误，也可能运行结果和预期不符。 猜猜看下面两种写法的输出结果。

# 正确写法
if [[ 1 == 2 ]]; then
    echo "imposible"
fi

# 错误写法
if [[ 1==2 ]]; then
    echo "imposible"
fi

利用(( ))来作为数值比较的condition

上面已经介绍了多种condition写法，这里再介绍一种写法，就是(( ))。(( ))内部可以进行数学运算及数值比较，它的退出状态是0或者非0（大多数情况下是1），可以用于是与非的条件判断。 (( ))命令支持下列比较操作符：'==', '!=', '>', '<', '>='，和'<='。

var=5
if((var>=2*1))
then
    echo "bingo"
fi

case语句

case语句可以达到和if/else相同的功能。但是针对分支过多的情况，使用case语句能够减少代码量，并且使代码更加清晰。下面伪代码展示了case语句的格式。

case word in
pattern1)
    command1;;
pattern2|pattern3)
    command2;;
*)
    command3;;
esac

其中pattern部分可以有多种形式。

  1. 用|符号来连接多个pattern，取逻辑或的含义
  2. pattern内部包含*符号作为通配符
  3. 采用*符号来作为pattern，匹配所有的情况(作default用)。

在将word与pattern进行匹配之前，会先对word进行扩展(~扩展，变量扩展，命令替换，数学计算)。

另外，case中可以做字符串的匹配

case "aa" in
aa)
    echo "this is aa"
    ;;
bb)
    echo "this is bb"
    ;;
cc|ee)
    echo "this is cc or ee"
    ;;
d*)
    echo "this is begin with d"
    ;;
*)  # default
    echo "unkown"
    ;;
esac

case $((3*4)) in
12)
    echo "12";;     # 输出12
13)
    echo "13";;
esac

case `echo aa` in
aa)
    echo "aa";;     # 输出aa
*)
    echo "other";;

case语句中每个匹配到的pattern下都有若干command，command的结尾可以有多种，分别有不同的含义

  1. ;;结尾符表示仅仅执行当前匹配到的command，执行完即结束
  2. ;&结尾符表示从当前匹配到的command开始执行，然后顺序执行下面所有的pattern对应的command，一直到case语句结束
  3. ;;&结尾符表示从当前匹配到的command开始执行，然后顺序执行下面能匹配成功的pattern对应的command，一直到case语句结束
case "aa" in
aa)
    echo "aa"
    ;&
bb)
    echo "bb"
    ;&
esac

case "aa" in
aa)
    echo "aa"
    ;;&
bb)
    echo "bb"
    ;;&
a*)
    echo "bb"
    ;;&
esac

select语句

select语句可以用来快速生成一个菜单，其语法类似于for语句(流程控制章节会讲)。select会为其后跟随的列表中的每一项分配一个序号，并在标准错误上输出序号和具体值的对应关系列表，并等待用户输入选择。如果用户输入的选择是序号中的某个，则如下代码所示，fname被设置的是用户选择的序号对应的值，而用户输入的序号则被存储在REPLY变量中。

# 列举当前目录下的所有文件，并且把用户的选择输出。注： 这里的*会被shell当做通配符，被替换为当前目录下的所有文件
select fname in *;
do
    echo you picked $fname \($REPLY\)
    break;
done

当用户输入完后，会执行do后面的命令，如果遇到break语句，则select执行完成，退出循环。否则会再次进入select语句，等待用户的新的输入。
